# Contributing

Thanks for your interest in improving Digital4Better Open Data.

This repository is primarily a data repository. Contributions are welcome when they improve the quality, coverage, traceability, or usability of the datasets.

## What Contributions Are Useful

Useful contributions include:

- fixing incorrect values
- adding missing records
- improving source traceability
- clarifying uncertainty or estimated values
- improving consistency across datasets
- improving public documentation such as the README

## What We Expect

When contributing data, please try to keep changes:

- sourced
- explicit
- consistent with the existing schema
- limited in scope when possible

If a value is estimated rather than officially disclosed, it should be clearly identifiable as such in the dataset whenever the schema supports it.

## Sources And Traceability

Good contributions usually come with one or more of the following:

- official documentation
- model cards
- technical reports or research papers
- public datasets from recognized organizations
- provider catalogs or specification pages

Please prefer primary sources when available. Secondary sources can still be useful, especially for synthesis or cross-checking, but they should not silently replace stronger evidence.

## Before Opening A Pull Request

Please make sure:

- the data you add or edit is relevant to the scope of the repository
- the format remains valid
- links to sources are still accessible
- naming stays consistent with nearby records
- the change is understandable from the pull request description

## Pull Request Tips

A good pull request usually explains:

- what dataset was changed
- why the change was needed
- which sources were used
- whether any values are estimated, inferred, or normalized

## Scope

This repository covers reference data used for topics such as:

- AI models
- cloud infrastructure
- geography and distances
- electricity mix and impact factors
- digital equipment reference data

If you are unsure whether a contribution fits the scope, opening an issue or a draft pull request is a perfectly good way to start the discussion.
